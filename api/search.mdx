---
title: 'POST /search'
description: 'Execute hybrid search queries combining BM25 and semantic vector search'
---

The `/search` endpoint performs hybrid search across indexed documents using a combination of BM25 full-text search (70%) and cosine similarity vector search (30%).

## Endpoint

```
POST /search
```

## Request Body

<ParamField path="query" type="string" required>
  The search query text to match against indexed documents
</ParamField>

<ParamField path="top_k" type="integer" default="10">
  Number of results to return. Must be a positive integer.
</ParamField>

## Response

Returns a JSON object containing an array of search results.

<ResponseField name="hits" type="array">
  Array of SearchHit objects ranked by hybrid score
  
  <Expandable title="SearchHit properties">
    <ResponseField name="doc_id" type="string">
      Unique document identifier
    </ResponseField>
    
    <ResponseField name="score" type="float64">
      Hybrid relevance score (0.7 * normalized_bm25 + 0.3 * normalized_cosine)
    </ResponseField>
    
    <ResponseField name="shard_id" type="string">
      Identifier of the shard containing this document
    </ResponseField>
    
    <ResponseField name="title" type="string">
      Document title field
    </ResponseField>
    
    <ResponseField name="text" type="string">
      Document text content
    </ResponseField>
  </Expandable>
</ResponseField>

## Hybrid Scoring Algorithm

The search endpoint uses a two-stage hybrid ranking approach:

1. **BM25 Candidate Retrieval**: Retrieves top 100 candidates using Bleve's BM25 full-text search
2. **Vector Reranking**: Computes cosine similarity between query and document embeddings (384-dimensional vectors)
3. **Score Fusion**: Combines normalized scores with weights:
   - 70% BM25 score (normalized by max BM25 score)
   - 30% cosine similarity (normalized from [-1, 1] to [0, 1])

<Note>
  The `rerankWindow` constant is set to 100, meaning BM25 retrieves 100 candidates before vector reranking. This balances retrieval quality with performance.
</Note>

## Examples

<CodeGroup>
```bash cURL
curl -X POST http://localhost:8081/search \
  -H "Content-Type: application/json" \
  -d '{
    "query": "machine learning algorithms",
    "top_k": 5
  }'
```

```python Python
import requests

response = requests.post(
    "http://localhost:8081/search",
    json={
        "query": "machine learning algorithms",
        "top_k": 5
    }
)

results = response.json()
for hit in results["hits"]:
    print(f"{hit['title']}: {hit['score']:.3f}")
```

```javascript JavaScript
const response = await fetch('http://localhost:8081/search', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
  },
  body: JSON.stringify({
    query: 'machine learning algorithms',
    top_k: 5
  })
});

const data = await response.json();
console.log(data.hits);
```

```go Go
package main

import (
	"bytes"
	"encoding/json"
	"net/http"
)

type SearchRequest struct {
	Query string `json:"query"`
	TopK  int    `json:"top_k"`
}

func main() {
	req := SearchRequest{
		Query: "machine learning algorithms",
		TopK:  5,
	}
	
	body, _ := json.Marshal(req)
	resp, _ := http.Post(
		"http://localhost:8081/search",
		"application/json",
		bytes.NewBuffer(body),
	)
	defer resp.Body.Close()
}
```

## Error Responses

<ResponseField name="400 Bad Request">
  Returned when the request body is malformed or cannot be parsed as JSON
</ResponseField>

<ResponseField name="500 Internal Server Error">
  Returned when embedding generation fails or the search index encounters an error
</ResponseField>
